# MONOCHROME — аудит и план развития

Дата аудита: 2026-07-14

## Что уже есть (сделано)

| Область | Состояние |
|---|---|
| Каркас | React 18 + Vite, роутинг, code splitting, LazyMotion (90.6 KB gzip) |
| Функциональность | Каталог с URL-фильтрами, карточка товара, корзина + localStorage, демо-чекаут |
| Медиа | 21 фото + 3 видео (Higgsfield), WebP-пайплайн (−42%), апскейл hero до 2K |
| Доступность | Focus trap + Esc в drawer, reduced motion, aria-атрибуты, alt-тексты |
| Качество | 19 unit-тестов (Vitest), CI на GitHub Actions (зелёный) |
| SEO-база | Favicon, per-page titles, OG-теги, meta description |
| Деплой | Конфиги Vercel + Netlify, SPA-fallback |

## Чего не хватает (аудит)

**Инженерное качество:** нет TypeScript, нет линтера/форматтера, нет E2E-тестов,
нет Error Boundary (ошибка рендера уронит всё приложение), нет страницы 404 для
несуществующих маршрутов (только для несуществующего товара).

**UX-пробелы:** нет избранного, нет поиска, нет зума фото на странице товара, нет
skeleton-заглушек при загрузке чанков/изображений, нет переходов между страницами,
select-фильтры — нативные (на мобильном ок, но выбиваются из эстетики).

**SEO/продакшн:** нет sitemap.xml/robots.txt, нет structured data (schema.org Product),
og:image указан относительным путём (соцсети требуют абсолютный URL — нужен домен
после деплоя), нет аналитики.

**Медиа:** hover-видео на карточках задумано, но не реализовано (нужно ~30 кредитов
Higgsfield, осталось ~18); у 8 товаров только один ракурс фото.

**Ручные шаги не выполнены:** PR #1 не смержен, репозиторий приватный, деплоя нет.

## План по фазам

### Фаза 0 — Публикация (15 минут, вручную, БЛОКИРУЕТ ВСЁ ОСТАЛЬНОЕ)

1. Смержить PR #1 на GitHub (CI уже зелёный).
2. Settings → Change visibility → Public (иначе портфолио никто не увидит).
3. vercel.com/new → Import `monochrome-store` → Deploy.
4. После деплоя: прописать полученный домен в `og:image` (абсолютный URL) и
   добавить live-ссылку в шапку README и в description репозитория.

**Результат:** проект существует как публичная витрина. Без этой фазы остальные не имеют смысла.

### Фаза 1 — Инженерная зрелость (1–2 вечера)

Самая ценная фаза для резюме джуна/мидла — именно это смотрят на код-ревью.

1. **ESLint (flat config) + Prettier** + скрипт `npm run lint`, добавить в CI.
2. **Миграция на TypeScript** (постепенно): `catalog.ts` → `CartContext.tsx` →
   компоненты → страницы. Строгий `tsconfig`, тип `Product` из JSON через `satisfies`.
3. **Error Boundary** вокруг маршрутов + маршрут `*` (страница 404 в стиле бренда).
4. **E2E-smoke на Playwright**: главная открывается → каталог фильтруется → товар
   добавляется в корзину → чекаут завершается. Один файл, ~5 сценариев, в CI.

**Результат:** типизированный, линтованный, покрытый E2E проект — уровень коммерческой разработки.

### Фаза 2 — UX-полировка магазина (1–2 вечера)

1. **Избранное**: сердечко на карточке, страница /favorites, localStorage
   (повторяет паттерн корзины — быстро).
2. **Лайтбокс/зум фото** на странице товара (клик — полноэкранный просмотр).
3. **Skeleton-заглушки** для изображений каталога (тёмный shimmer, пока грузится WebP).
4. **Переходы между страницами** через AnimatePresence (fade, 200 мс — кинематографично,
   но не мешает).
5. **Стилизованные фильтры**: кастомный dropdown вместо нативного select
   (headless-подход, с клавиатурной навигацией — заодно демонстрирует a11y-скилл).

**Результат:** сайт ощущается как продукт, а не демо.

### Фаза 3 — Медиа-апгрейд через Higgsfield (когда пополнятся кредиты, ~40–60 кредитов)

1. **Hover-видео карточек новинок**: 4 микро-видео по 2–3 с (image-to-video от
   существующих фото, kling3_0_turbo, ~30 кредитов) — карточка оживает при наведении.
2. **Вторые ракурсы** для 8 товаров с одним фото (nano_banana_pro, ~16 кредитов).
3. **Видео на странице товара** для 2–3 флагманских позиций (ткань в движении).
4. Опционально: **эмбиент-звук для lookbook** (generate_audio, кнопка mute/unmute).

**Результат:** максимальный «вау» — то, ради чего в проекте Higgsfield.

### Фаза 4 — Продакшн-симуляция (1 вечер)

1. **schema.org Product JSON-LD** на страницах товаров (rich snippets).
2. **sitemap.xml + robots.txt** (генерация из products.json на этапе сборки).
3. **Аналитика** Plausible/Umami (privacy-friendly, бесплатно) — событие add_to_cart.
4. **Lighthouse-прогон** после деплоя, цель 95+ по всем категориям; починить найденное
   (скорее всего: preload hero-poster, fetchpriority, font-display).

**Результат:** можно честно писать в резюме «оптимизировал до Lighthouse 95+».

### Фаза 5 — Опциональный «вау» (если захочется выделиться)

- **AI-стилист**: чат «собери образ» — на статичных правилах поверх products.json,
  либо через Claude API (тогда нужен серверless-эндпоинт на Vercel Functions —
  заодно первый бэкенд-опыт в проекте).
- **i18n EN/UK** через react-i18next — востребованный навык для украинского рынка.
- **View Transitions API** для переходов каталог→товар (морфинг картинки) —
  свежая технология, хорошо смотрится в демо.
